### Context

사용자의 메모를 바탕으로 n8n을 활용하여 첫 워크플로우 자동화를 구축하는 과정을 기록합니다. 사용자는 디스코드 메시지 수신을 트리거로, 에이전트 실행, TIL 형식 변환, GitHub 푸시, 인덱스 추가, 완료 메시지 발신까지 이어지는 자동화 파이프라인을 구상했습니다.

### Core

아래는 사용자의 요구사항을 기반으로 n8n 워크플로우를 구성하는 단계별 설명입니다. 각 단계별로 필요한 노드와 설정을 명시합니다. (실제 n8n 워크플로우 JSON은 복잡하여 포함하지 않으며, 개념적인 설명으로 대체합니다.)

*   **트리거 노드**: `Discord Trigger` 노드를 사용하여 특정 채널의 메시지 수신을 감지합니다.
*   **에이전트 실행 노드**: 수신된 메시지를 처리할 에이전트(LLM 등)를 호출하는 노드를 구성합니다. 이는 `HTTP Request` 노드를 통해 외부 API를 호출하거나, n8n 내에서 Python 스크립트를 실행하는 `Code` 노드를 사용할 수 있습니다.
*   **TIL 형식 변환 노드**: 에이전트가 생성한 응답을 TIL(Today I Learned) 형식(.md 파일)으로 변환하는 로직을 구현합니다. 이 역시 `Code` 노드를 활용하여 Python 스크립트로 처리하는 것이 일반적입니다.
*   **GitHub 푸시 노드**: 변환된 TIL 파일을 GitHub 저장소에 푸시하는 노드를 설정합니다. `GitHub: Create File` 또는 `GitHub: Commit` 등의 노드를 사용할 수 있습니다. GitHub API 인증 정보(토큰) 설정이 필요합니다.
*   **인덱스 추가 노드**: 푸시된 TIL 파일 목록을 관리하는 인덱스 파일을 업데이트합니다. 이는 `File` 노드를 사용하여 기존 인덱스 파일을 읽고, 새 내용을 추가한 후 다시 저장하는 방식으로 구현할 수 있습니다.
*   **완료 메시지 발신 노드**: 모든 과정이 성공적으로 완료되었음을 알리는 메시지를 디스코드로 발송합니다. `Discord: Send Message` 노드를 사용합니다.

### Insight

n8n을 처음 사용해 보았는데, 직관적인 인터페이스와 다양한 노드 연동을 통해 복잡한 워크플로우를 비교적 쉽게 구축할 수 있다는 점이 매우 인상 깊었습니다. 특히 코드 없이도 상당 부분의 자동화를 구현할 수 있어 생산성 향상에 크게 기여할 것으로 기대됩니다. 다만, 복잡한 로직이나 외부 API 연동 시에는 `Code` 노드를 통한 스크립트 작성이 불가피하며, 이때는 Python 등 프로그래밍 지식이 요구됩니다.
