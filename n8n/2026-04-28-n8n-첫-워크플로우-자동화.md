### Context

첫 n8n 워크플로우 자동화를 성공적으로 구축했습니다. 이 자동화는 디스코드 메시지 수신을 트리거로 사용하여, Agent 실행, TIL 형식 변환, GitHub 푸시, 인덱스 추가, 완료 메시지 발송까지의 전체 과정을 자동화합니다.

### Core

*   **트리거**: 디스코드에서 메시지 수신 시 워크플로우 시작
*   **Agent 실행**: 수신된 메시지를 기반으로 Agent 로직 실행
*   **TIL 형식 변환**: Agent 실행 결과를 TIL(Today I Learned) 형식의 마크다운 문서로 변환
*   **GitHub 푸시**: 변환된 TIL 문서를 지정된 GitHub 저장소에 푸시
*   **인덱스 추가**: 푸시된 TIL 문서에 대한 인덱스 업데이트 (예: README 파일 수정)
*   **완료 메시지 발송**: 모든 작업 완료 후 디스코드 채널에 완료 메시지 알림

### Insight

n8n을 처음 사용해보았는데, 직관적인 인터페이스와 다양한 노드 연동 기능 덕분에 복잡한 워크플로우를 코딩 없이도 손쉽게 구축할 수 있다는 점이 매우 인상 깊었습니다. 특히 디스코드 챗봇 연동부터 GitHub 자동 푸시까지, 여러 서비스를 연결하는 과정이 n8n 노드 몇 개로 구현 가능하다는 점은 생산성 향상에 크게 기여할 것으로 기대됩니다.
