### Context
1인 게임 개발자는 기획, 아트, 프로그래밍, 사운드 등 모든 분야를 혼자 관리해야 하므로 '범위 관리(Scope Management)'와 '효율적인 파이프라인 구축'이 필수적입니다. 특히 2026년 기준 Unity 6와 같은 최신 엔진 환경에서는 AI 도구를 활용하여 개발 시간을 단축하고 기술적 장벽을 낮추는 전략이 생존의 핵심입니다. 이 문서는 1인 개발자가 프로젝트를 완수하기 위해 반드시 준수해야 할 기술적 워크플로우를 정리합니다.

### Core
유니티 1인 개발 최적화 파이프라인과 기술 스택은 다음과 같습니다.

* 프로젝트 규모 산정 (MVP 전략)
  * 최소 기능 제품(Minimum Viable Product) 정의: 핵심 재미 요소 하나에 집중하여 3개월 내 프로토타입 완성
  * 가변적 스코프(Scope): 개발 중 핵심 로직 외의 부가 기능은 과감히 삭제하거나 에셋 스토어 활용

* AI 기반 에셋 및 코드 워크플로우
  * Unity Muse: 텍스트 및 스케치를 통한 텍스트 제작, 텍스트 기반 애니메이션 생성 및 스프라이트 생성 활용
  * Unity Sentis: 온디바이스(On-device) AI 모델 통합을 통해 복잡한 NPC 행동 트리나 실시간 이미지 처리를 신경망 모델로 대체
  * GitHub Copilot / Cursor: C# 스크립팅 시 보일러플레이트 코드 작성 및 리팩토링 자동화

* 버전 관리 및 협업(나와의 협업)
  * Git LFS(Large File Storage): 고용량 텍스트 및 3D 모델 관리 필수
  * 유니티 클라우드(Unity Cloud): 빌드 자동화 및 대시보드를 통한 리소스 관리 효율화

* 코드 아키텍처 및 최적화
  * 컴포넌트 기반 설계: 재사용 가능한 모듈형 스크립트 작성으로 다른 프로젝트로의 이식성 확보
  * 스크립터블 오브젝트(Scriptable Objects): 데이터와 로직을 분리하여 기획 데이터 수정 시 재컴파일 시간 단축

### Insight
1인 개발의 가장 큰 적은 '완벽주의'가 아닌 '중도 포기'입니다. 기술적으로는 Unity 6의 향상된 렌더링 성능(URP/HDRP)보다 Unity Muse나 Sentis 같은 AI 도구를 얼마나 숙달하여 제작 공수를 줄이느냐가 프로젝트 완수 가능성을 결정합니다. 특히 Sentis를 활용하면 클라우드 비용 없이도 고성능 AI 기능을 게임 내에 구현할 수 있어 비용 관리가 중요한 1인 개발자에게 큰 이점을 제공합니다. 기술 검증 시에는 최신 안정화 버전(LTS)을 사용하고, 새로운 AI 기능을 도입할 때는 반드시 작은 샌드박스 프로젝트에서 먼저 테스트한 뒤 메인 프로젝트에 병합하는 보수적인 접근이 필요합니다.
